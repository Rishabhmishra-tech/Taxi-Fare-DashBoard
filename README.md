# 🚖 Taxi Fare Dashboard – Data Analysis Project

## 📌 Project Overview
The Taxi Fare Dashboard project analyzes taxi trip data to understand fare patterns, trip distances, passenger trends, and peak demand times.  
The goal of this project is to perform data cleaning, exploratory data analysis (EDA), and visualization to generate insights that help improve taxi service efficiency and pricing strategies.

This project demonstrates practical skills in data analysis, data visualization, and dashboard creation.

---

## 🎯 Objectives
- Analyze taxi trip data to identify fare patterns.
- Understand how distance, time, and passengers affect taxi fares.
- Identify peak hours and high-demand locations.
- Build an interactive dashboard to visualize taxi trip insights.

---

## 📂 Dataset
The dataset contains taxi trip records with features such as:

- Pickup Date & Time  
- Dropoff Date & Time  
- Passenger Count  
- Trip Distance  
- Pickup Location  
- Dropoff Location  
- Fare Amount  
- Tip Amount  
- Total Amount  

---

## 🔄 Project Workflow

### 1️⃣ Data Collection
Taxi trip dataset collected from public sources.

### 2️⃣ Data Cleaning
- Handle missing values
- Remove incorrect fare values
- Convert date-time columns
- Filter outliers

### 3️⃣ Exploratory Data Analysis (EDA)
- Fare distribution analysis
- Trip distance vs fare analysis
- Passenger count analysis
- Peak demand hours analysis

### 4️⃣ Data Visualization
The following visualizations were created:
- Total Trips
- Average Fare
- Fare by Distance
- Trips by Hour
- Passenger Count Distribution
- Pickup Location Analysis

### 5️⃣ Dashboard Creation
An interactive dashboard was built to display key insights such as:
- Total trips
- Average fare
- Peak travel hours
- Distance vs fare trends
- Passenger distribution

---

## 📊 Key Insights
- Taxi fares increase as trip distance increases.
- Most trips occur during morning and evening rush hours.
- Most taxi rides include 1–2 passengers.
- Some pickup locations have significantly higher demand.
